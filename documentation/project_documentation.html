1. Play the Web

1.1 Description

Play the Web is a multi-user educational game for learning the basic concepts regarding Web technologies in a fun and captivating way. The concepts that can be learnt using this application are HTML, CSS, Web server, Cookies, PHP and JavaScript.

1.2 Context of Use

The context of use is wanting to learn and understand the basics about Web technologies in a fun way that does not feel like studying and without taking real-life classes.

1.3 Main Audience

The main audience consists primarily of people that have an interest in Web technologies and who don’t attend any kind of classes in the field, especially young students that may think of enrolling to a Computer Science Faculty.

2. Design Patterns

2.1 Mobile vs. Desktop

The application has been designed as both a mobile application and a desktop application.

2.1.1 Mobile UI Patterns

The patterns used for the mobile version of the application are:

  • Navigation:
    • Bottom Bar: the application has a bottom bar with the most important icons: “trophy” the achievements page, “house” for home page, “chat boxes” for the forum page and the “profile” icon.
  • Actions and commands:
    • Tap: all the buttons work by simply tapping them once.
    • Swipe Left-Right: for the pages that have the “left arrow” icon in the right-up corner, there is the swipe left-right option to change the screen to the previous page
  • Tables and Lists:
    • Tables: the “Challenge” page uses a table for showing the name of the concept and the best time in which that particular challenge was completed
  • Form Patterns:
    • Input Fields: the application uses various kinds of input; in the “Account Information” page, there are: text input for “Username” and “Password”, email input, phone number input and image input for “Profile Picture”; in the “Reset Password” page, there is number input for the verification code
    • Field Labels: all the input fields have suggestive labels: the field labelled “Email” in the “Sign up” page asks for email input; the field labelled “Name” asks for text input
    • Progress Bar: the “Home” page has a progress bar for each individual concept showing how much of the actual game has been completed
    • Chart: the page “Statistics” has a chart displaying the activity of the user

2.1.2 Desktop UI Patterns

The patterns used for the Desktop version of the app are:

  • Navigation:
    • Menu Bar: all the pages of the app have the navigation bar on the top part of the screen.
  • Layout:
    • Center Stage: on all the pages, the part that offers the functionalities is set in the center of the screen.
  • Actions and commands
    • Tap: all the possible actions happen only by clicking on them using the mouse.

2.2 Conversational Patterns

The application has a special forum page intended to people for communicating. This page allows users to engage in discussion on certain topics that were chosen before or to start their new conversation by asking a question.

The pattern used are:

  • Sender Message: the user can either send messages/ comments on a discussion set by somebody else or can send their own message/ start a new conversation for others to join in
  • Receiver Message: the user receives to all the comments/ messages in any conversation they choose to take part in
  • Unsend/ Delete Message: the user can delete any message/ comment written by them
  • Upload Media: in the writing bar, there is the “paper clip” icon that gives the option to add various kinds of media like pictures, videos or pdf files
  • Add emoji/ reaction: in the writing bar, the “emoji” icon allows the user to send various emojis

3. Application

3.1 Metaphors/ Idioms

All the icons used in this application are metaphors (the “chat boxes” icon for the “Forum” page, the “trophy” icon for the “Achievements” page) or idioms (the “triangle” meaning “play” which is a popular association among games, the “pen” icon for changing information like email) that are very easy to understand, as they are commonly used in applications.

3.2 Affordances

The application is built such that it is easy to understand without having lots of previous experience in working with such an app. However, in case there is still need for explanation, the user may go on the “Settings” page and click on “help”. They will be redirected to a “Help” page, where the only need to select the specific topic they are looking for.

3.3 User Flow

The application opens on the “Home” page. From here, the user has multiple possible ways to follow.

  1. The app opens on the “Home” page. The user can always come back to this page by clicking on the “home” icon on the bottom bar (for the mobile app) or the “home” box on the navigation bar (for the desktop version) Here, there are presented the different concepts that may be tested, along with the percentage of progress that has already been made. By clicking on the “triangle” icon beside a concept, the user is redirected to the page where they can play and learn.
  2. On the right-up corner, there is the “settings” icon that redirects the user to the “Settings” page. Here, they can change the language, time zone and the light/ dark theme. The user can also log out by clicking on the “Log out” button. There are also given information about the legal terms, policy and open-source libraries.
  3. If the user is not yet logged in, they are redirected to the “Sign in/ up” page, where they can login, if they already have an account, or can register by creating a new account.
  4. The last section of the “Settings” page is the line “Need help?”. Here, by clicking on the word “help”, the user is redirected to a new page. The user can get information about a certain topic by clicking on the button labelled that way.
  5. The “Profile” page, that can be accessed by clicking the “profile” icon on the bottom bar or the “profile” box on the navigation bar, displays the current “Total progression” made by the user. Here, by clicking on the button labelled “Statistics”, the user is redirected to a page that displays a chart of time spent using this app. The user can choose if the time displayed refers to this week only or to the whole month.
  6. Back on the “Profile” page, by clicking on the button labelled “Account info”, the user is redirected to a page displaying their personal info: username, email, phone number and profile picture. Each of these fields have a “pen” icon attached that, by clicking, allows the user to modify the information. On the bottom of the page, there is a button labelled “Reset password” that redirects the user to a new page.
  7. The “Reset Password” page requires a verification code for security purposes and states the question whether the code should be sent via email or phone; the user chooses one of the two by clicking on the either on the “envelope” icon, meaning message, or the “G” icon. After the code has been introduced, the user has to click on the “Validate” button and, if the code is correct, they are redirected to a “New Password” page. Here, they need to type in the new password, confirm it by typing it again and then press either the “Save” button, or the “Cancel” button in case they changed their mind.
  8. The page “Forum” can be accessed clicking on the “chat boxes” icon or the “Forum” box. This page presents a list of discussion topics and, by clicking on any of them, the user is redirected to the conversation taking place on that topic. On this new page, the user can read all the replies given by other users and can also reply in their own comment. The icon in the left-up corner, the “left arrow”, takes the user back to the “Forum” page.
  9. Still on the “Forum” page, there is a button labelled “Create Subject” that redirects the user to a new page. Here, they need to type in a title for the new discussion they want to start, select one of the given categories and write their full-length question. Then, they may cancel the conversation by clicking on the “Cancel” button or can proceed and post the discussion by pressing the “Submit” icon.
